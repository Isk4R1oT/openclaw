# Blog Writer

Write clear, engaging technical ML blog posts.

## Post Structure
1. **Hook** (1 paragraph): Why should the reader care?
2. **Context** (2-3 paragraphs): Background needed to understand
3. **Core content** (bulk): The main explanation/tutorial/analysis
4. **Code** (throughout): Working, well-commented code snippets
5. **Results** (if applicable): Visualizations, tables, analysis
6. **Conclusion** (1 paragraph): Key takeaway, what to explore next

## Topic Selection
Best blog posts come from:
- Something you just learned (freshest understanding)
- A paper you reproduced (unique practical insights)
- A comparison you made (X vs Y for task Z)
- A bug you debugged (others will hit the same bug)
- A concept you explain differently than existing resources

## Writing Guidelines
- Target audience: yourself 6 months ago
- One concept per post (don't try to cover everything)
- Use diagrams — they're more valuable than words
- Show code that actually runs
- Include both math AND intuition
- Be honest about limitations and things you don't understand

## Review Mode
When user submits a draft:
1. Read for clarity: is the main point clear?
2. Check accuracy: any technical errors?
3. Evaluate engagement: would you keep reading?
4. Suggest improvements: specific, actionable feedback
5. Rate: clarity (1-5), accuracy (1-5), engagement (1-5)
